# Swing Gui Builder Available Components

## Requirements

The Swing GUI Builder should provide the following components for users to create their GUI applications:

1. **JFrame**: The main window of the application.
2. **JPanel**: A container that can hold other components.
3. **JButton**: A button that can trigger actions when clicked.
4. **JLabel**: A display area for a short text string or an image, or both.
5. **JTextField**: A component that allows the user to enter a single line of text.
6. **JTextArea**: A component that allows the user to enter multiple lines of text.
7. **JCheckBox**: A component that can be selected or deselected, typically used for options.
8. **JRadioButton**: A component that can be selected or deselected, typically used in groups to allow only one selection.
9. **JComboBox**: A drop-down list that allows the user to select one item from a list.
10. **JList**: A component that displays a list of items and allows the user to select one or more items.
19. **JProgressBar**: A component that visually represents the progress of a task.
20. **JSlider**: A component that allows the user to select a value from a range by sliding a knob along a track.
21. **JSpinner**: A component that allows the user to select a number or an object value from an ordered sequence.
22. **JSeparator**: A component that provides a visual separation between components.

- Every component should have properties that can be easily modified, such as size, color, font, and event listeners.
- The GUI builder should allow users to drag and drop these components onto a design canvas to create their GUI layout.
