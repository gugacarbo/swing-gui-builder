# Subtask 04 - Revisar integracao do schema do projeto

## Objetivo

Garantir que `.swingbuilder.json` tenha validacao e autocomplete funcionando de forma confiavel.

## Arquivos principais

- `package.json`
- `schemas\swingbuilder.schema.json`

## Escopo

- revisar a forma como o schema e exposto pela extensao
- ajustar a contribuicao de schema para o formato suportado pelo VS Code
- validar o comportamento com a extensao empacotada ou instalada

## Fora de escopo

- ampliar o schema com novas features ainda nao implementadas

## Criterio de pronto

- o arquivo `.swingbuilder.json` recebe validacao de estrutura
- propriedades conhecidas aparecem com autocomplete e descricoes
- caminhos e URIs do schema funcionam fora do ambiente de desenvolvimento
